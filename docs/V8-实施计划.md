## 一、产品定位

**用户输入自然语言 → 系统返回数字 + 图表 + 原文引用的分析报告。**

### 核心流程

```
用户上传文档(年报/研报/尽调)
  → loader 解析表格+正文
  → 表格数字 → 规则提取 → SQL (100% 准确)
  → 正文 → RAG 索引 (可溯源引用)
  → Agent: Planner → Executor → Reporter
  → 数字 + 解读 + 图表 + 引用 = 完整报告
```

### 双入口

| 入口 | 数据来源 | 用途 |
|------|---------|------|
| **预设标签**（3-5家） | AKShare + 年报PDF | 打开即体验，面试演示 |
| **自定义上传** | 用户自己的 PDF/Word/MD | 核心使用场景 |

### 关键设计决策

1. **数字绝不走 LLM**：表格→规则提取→SQL，100% 准确
2. **RAG 提供溯源**：引用原文，不提取数字
3. **SQL+RAG 互补**：SQL 答"多少"，RAG 答"为什么"
4. **评测诚实**：区分能力内/外，不过拟合
