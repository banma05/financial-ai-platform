## 二、Agent 执行流程（核心链路）

```mermaid
sequenceDiagram
    participant U as 用户
    participant P as Planner (LLM)
    participant E as Executor
    participant SQL as SQL数据层
    participant RAG as RAG引擎
    participant C as Calculator
    participant CH as Chart
    participant RP as Reporter (LLM)

    U->>P: "分析茅台2024盈利能力"
    P->>P: LLM拆解 → 4个子任务
    P->>E: 任务列表（按依赖排序）

    E->>SQL: task1: 查营收/净利/ROE
    SQL-->>E: {营收:1708.99, 净利:893.35, ROE:34.2%} (5ms)

    E->>C: task2: 计算毛利率/净利率
    C-->>E: {毛利率:92.3%, 净利率:52.3%} (1ms)

    E->>CH: task3: 生成盈利能力图表
    CH-->>E: <base64柱状图> (0.5s)

    E->>RAG: task4: 辅助解读"毛利率变化原因"
    RAG-->>E: "原材料成本上涨+渠道改革..." (2s)

    E->>RP: 聚合全部结果 → LLM生成报告
    RP-->>U: Markdown报告 + 图表 (3s)

    Note over U,RP: 端到端 < 8秒，数字100%准确
```

## 三、数据查询双引擎

```mermaid
graph LR
    Q["用户查询<br/>'茅台2024 ROE'"] --> ROUTER{查询路由}

    ROUTER -->|"查数字<br/>(ROE/毛利率/营收)"| SQL_ENGINE["SQL 引擎<br/>指标匹配 → SQL查询<br/>→ 毫秒级 · 100%准确"]
    
    ROUTER -->|"查原因<br/>(为什么下降/管理层怎么说)"| RAG_ENGINE2["RAG 引擎<br/>BM25+语义+LambdaMART<br/>→ 检索MD&A章节<br/>→ LLM解读"]

    SQL_ENGINE --> MERGE["结果聚合"]
    RAG_ENGINE2 --> MERGE
    MERGE --> RESPONSE["完整答案<br/>数字+解读+溯源"]
```

## 四、技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| **LLM** | DeepSeek v4-flash / v4-pro | flash=简单提取，pro=复杂推理+报告 |
| **Embedding** | BAAI/bge-base-zh-v1.5 (768维) | 本地免费，中文SOTA |
| **Reranker** | BAAI/bge-reranker-v2-m3 | 本地免费 |
| **向量库** | ChromaDB (HNSW) | 轻量免运维 |
| **分词** | jieba + 138财务术语词典 | 精确模式 |
| **Agent框架** | LangGraph StateGraph | 业界标准编排 |
| **数据源** | AKShare + SQLite | A股免费数据 |
| **后端** | FastAPI + SSE | 流式响应 |
| **前端** | Streamlit | 快速原型 |
| **容器化** | Docker + docker-compose | 生产部署 |
