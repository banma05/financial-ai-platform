### 战略定位

**不在"数据量"和"功能数"上跟同花顺拼。专注做"真正会做基本面分析"的 AI。**

四条核心能力：
1. **数据准确可溯源** — 每条数字都能追溯到原始财报（对标同花顺）
2. **因果关系推断** — 不只告诉你"毛利率降了"，还告诉你"为什么降"（差异化）
3. **个性化分析框架** — 适配不同用户的分析风格（竞品短板）
4. **交互式可视化** — 图表可交互、可钻取、有意义（当前 0%→100%）

## 剩余工作（按优先级）

### P0 — 数据准确性地基（先做）

#### 1. ✅ 金融公司数据补填 (已完成 07-22)
**问题表象**: 招商银行/中国平安/平安银行/中信证券 4 家 `net_profit_attr_parent` / `equity_attr_parent` 为 NULL

**根因**: `financial_populator.py` 的 `METRIC_NAME_MAP` 只有工业企业的 AKShare 列名映射。
银行/保险/券商使用不同的中文列名（如"归属于母公司的净利润" vs "归属于母公司所有者的净利润"），
全部匹配失败 → 静默丢弃（line 197 `if metric_key is None: continue`）。
这不是某几家公司的 bug，是数据管道对金融行业的系统性遗漏。

**修复（三层防御）**:
1. **源头修** `financial_populator.py`: `METRIC_NAME_MAP` 补 14 个金融企业列名变体 + 未映射字段告警
2. **查询兜底** `financial_query.py`: `_KEY_FALLBACK` 通用回退引擎（simple + computed 两级）
3. **存量修复** `scripts/backfill_financial_data.py`: 126+84 条补填（适合不改动存量数据时用）
- 文件: `db/financial_populator.py`, `db/financial_query.py`, `scripts/backfill_financial_data.py` (新建)

**验证**: 4家金融公司 ROE/净利率 全部从"无数据"→正常计算，与年报偏差 <2%。385测试全绿。

#### 2. ✅ PB/EBITDA 计算修复 (已完成 07-22)
**修复**:
- `financial_calc.py`: 策略5 bvps自动推算 + 策略6 ebitda←operating_profit回退
- `financial_query.py`: 新增 EBITDA/EBITDA率 复合指标别名
- 文件: `agent/tools/financial_calc.py`, `db/financial_query.py`
**验证**: 茅台PB=8.65, 招商PB=0.87, EBITDA率=53.04% ✅

#### 2.5 ✅ 数据质量基础设施 (新增, 已完成 07-22)
**问题**: 数据无可溯源、图表白屏、RAG形同虚设、报告无法审查
**修复**:
- 数据溯源: `financial_query.py` 每个数据点标注来源(sql/fallback/computed)+置信度
- 图表零空白: `chart.py` ≤1数据点自动降级skip+说明, 中英文键去重
- RAG主力化: `planner.py` 6个模板全补rag_context(之前仅1个)
- 报告可审查: `reporter.py` 新增"数据可靠度"章节 + 执行摘要格式
- 数据流架构: `executor.py` 依赖注入分两路(display_name直注+原始键ParamInjector)
- 数值校验: `reporter.py` 三层智能化(去单位后缀+推导检测+白名单), 消除误报
- 文件: `db/financial_query.py`, `agent/tools/chart.py`, `agent/planner.py`,
  `agent/reporter.py`, `agent/executor.py`, `agent/schemas.py`, `agent/tools/financial_calc.py`
**验证**: 茅台全模板53/53 OK, 图表3数据点柱状图, 报告无数值校验误报 ✅

#### 3. 评测体系重建 (3-4h) — ⚠️ 当前最高优先级
**问题**: 15 模板题 + 循环论证 = 假 100%
当前评测题直接使用系统数据库中的值做 required_numbers，数据库里有什么评测就"期望"什么 → 永远不会错。
**修复**:
- 扩充至 50 题：30 题模板覆盖 + 20 题自由拆解
- 每道题标注 `required_numbers`（从真实年报人工验证，非数据库读取）
- 新增评分维度：数据溯源率、图表渲染率、行业基准对比
- 带自由拆解场景的评测（"分析比亚迪2024年财务整体表现" 等）
- **本次新增**: 评测结果按公司分离（有RAG的5家 vs 仅有SQL的15家），避免知识库覆盖不足干扰评分
- 文件: `evaluation/data/agent_questions_v9.json` (新建), `evaluation/agent/bench_agent.py`

- `cross_company_profit` 模板: 验证两公司数据是否都成功查询
- 文件: `db/financial_query.py`, `agent/tools/data_query.py`

### P3 — 长期规划（架构演进）

#### 10. 数据库 SQLite → PostgreSQL
- 当前 25 家公司、7MB → 扩展到 5000 家时必须迁移
- EAV 模式在 PG 上性能更好（JSONB 替代 EAV 或加分区表）

#### 11. RAG 数据量扩充 — ⚠️ 当前仅6家公司，14家无RAG解读
**现状**: 知识库仅覆盖 贵州茅台/腾讯控股/比亚迪/宁德时代/五粮液 + 测试文档（14份）
  数据库已有20家公司，其中14家（含全部金融企业招商/平安/中信等）无任何RAG文档
**影响**: 非茅台/腾讯/比亚迪/宁德/五粮液的分析报告缺少"原文解读"章节，
  rag_context 检索到无关公司年报（查招商银行返回腾讯内容）
**修复**:
- 优先补金融企业（招商银行/中国平安/平安银行/中信证券）各至少1份年报
- 其次补大市值公司（美的/格力/恒瑞等）各1份年报
- 当前 14 文档/5000 chunks → 近期目标 30+文档/15K chunks

#### 12. Agent 验证节点
- 在 Executor 后加 `verify_node`
- 数据不足时回退 Planner 重新规划
- 置信度驱动报告生成（低置信度数据标注）

#### 13. 多轮对话支持
- 会话上下文管理器（记录当前公司、指标集）
- 增量执行引擎（追加指标时不重跑整个分析）
