### 战略定位

**不在"数据量"和"功能数"上跟同花顺拼。专注做"真正会做基本面分析"的 AI。**

四条核心能力：
1. **数据准确可溯源** — 每条数字都能追溯到原始财报（对标同花顺）
2. **因果关系推断** — 不只告诉你"毛利率降了"，还告诉你"为什么降"（差异化）
3. **个性化分析框架** — 适配不同用户的分析风格（竞品短板）
4. **交互式可视化** — 图表可交互、可钻取、有意义（当前 0%→100%）

## 剩余工作（按优先级）

### P0 — 数据准确性地基（先做）

#### 1. 金融公司数据补填 (1-2h)
**问题**: 招商银行/中国平安/平安银行/中信证券 4 家 `net_profit_attr_parent` 为 NULL
**修复**:
- 写脚本查 DB 中 `net_profit` 是否有值 → 有则复制到 `net_profit_attr_parent`
- `financial_query.py`: ROE/净利率 公式加 `net_profit_attr_parent → net_profit` 回退链
- 文件: `scripts/backfill_financial_data.py` (新建), `db/financial_query.py`

#### 2. PB/EBITDA 计算修复 (1-2h)
**问题**: PB 缺 bvps、EBITDA 缺 ebitda 参数
**修复**:
- `financial_calc.py`: `_auto_fill_params` 策略5 加 try/except 兜底
- 当 bvps 推导失败时，用 `equity / total_shares` 估算（从 EPS 反推总股本）
- `financial_query.py`: 加 `EBITDA` 复合指标别名（用 operating_profit + depreciation 近似）
- 文件: `agent/tools/financial_calc.py`, `db/financial_query.py`

#### 3. 评测体系重建 (3-4h)
**问题**: 15 模板题 + 循环论证 = 假 100%
**修复**:
- 扩充至 50 题：30 题模板覆盖 + 20 题自由拆解
- 每道题标注 `required_numbers`（从真实年报人工验证）
- 新增评分维度：数据溯源率、图表渲染率、行业基准对比
- 带自由拆解场景的评测（"分析比亚迪2024年财务整体表现" 等）
- 文件: `evaluation/data/agent_questions_v9.json` (新建), `evaluation/agent/bench_agent.py`

### P1 — 能力补全（再做）

#### 4. 新增 4 个核心模板 (3-4h)
**估值分析** (valuation): PE/PB/PS + 历史估值分位 + 行业对比
**营运能力** (operating): 存货/应收/总资产周转率
**行业对比** (industry_compare): 公司与行业均值对比雷达图
**费用结构** (cost_analysis): 四费（销售/管理/研发/财务）占比趋势
- 文件: `agent/planner.py`

#### 5. 跨公司对比 SQL 支持 (2-3h)
**问题**: `parse_query` 只取第一家公司的 symbol
**修复**:
- `parse_query`: 多公司返回 `List[str]` symbols
- `try_query`: 遍历 symbols，返回以公司前缀的 key（如 `茅台_营收_2024`）
- `data_query.py`: LLM 提取时区分多公司数据
- 文件: `db/financial_query.py`, `agent/tools/data_query.py`

#### 6. 报告专业性升级 (3-4h)
**问题**: 摘要 = 任务清单，不是执行摘要
**修复**:
- 重写 `_build_summary`: 执行摘要（Executive Summary）
- `_generate_insights`: 解除"禁止判断"过度约束 → "数据驱动判断+溯源标注"
- 行业感知风险评估: 行业基准阈值表
- 数据概览加单位列
- 文件: `agent/reporter.py`

### P3 — 长期规划（架构演进）

#### 10. 数据库 SQLite → PostgreSQL
- 当前 25 家公司、7MB → 扩展到 5000 家时必须迁移
- EAV 模式在 PG 上性能更好（JSONB 替代 EAV 或加分区表）

#### 11. RAG 数据量 10 倍扩充
- 当前 14 文档/5000 chunks → 目标 100+文档/100K chunks
- 覆盖：年报（每年每公司）+ 招股书 + 行业研报 + 宏观数据

#### 12. Agent 验证节点
- 在 Executor 后加 `verify_node`
- 数据不足时回退 Planner 重新规划
- 置信度驱动报告生成（低置信度数据标注）

#### 13. 多轮对话支持
- 会话上下文管理器（记录当前公司、指标集）
- 增量执行引擎（追加指标时不重跑整个分析）
